## Synthese

Le site dispose deja d'un socle Drupal solide et de plusieurs pages services :

- `/agence-drupal-belgique`
- `/creation-site-drupal`
- `/refonte-site-drupal`
- `/migration-drupal`
- `/maintenance-drupal`
- `/audit-drupal`
- `/accessibilite-seo-optimisation`
- `/ia-integree`
- `/ia-drupal` et les cas d'usage IA associes

Le prochain enjeu n'est donc plus seulement de produire des pages Drupal. Il est
d'ouvrir l'acquisition SEO a des recherches plus larges, sans perdre la
credibilite technique.

Positionnement recommande :

> E-merging Digital accompagne PME, ASBL et institutions dans la creation, la
> refonte et l'evolution de sites web professionnels, avec une expertise senior
> en PHP, Drupal, Symfony, Laravel, SEO technique et IA encadree.

Cette phrase doit guider les futurs contenus, mais chaque page doit rester
precise : une intention, un public, une promesse, une preuve, un CTA.

## Changement strategique

### Ancienne perception a corriger

Le site peut etre percu comme :

- une agence Drupal ;
- un specialiste IA dans Drupal ;
- un prestataire technique pour projets deja qualifies.

Cette perception est utile, mais trop etroite. Elle rate les prospects qui
cherchent :

- un site professionnel ;
- une refonte ;
- une agence web senior ;
- un developpement web sur mesure ;
- une aide PHP/Symfony/Laravel ;
- une automatisation IA prudente ;
- une meilleure visibilite SEO.

### Nouvelle perception cible

Le site doit etre compris comme :

- une agence web senior, pas une agence generaliste interchangeable ;
- une equipe PHP capable d'arbitrer CMS, framework et integrations ;
- un partenaire Drupal quand le contenu, le multilingue et la gouvernance
  comptent ;
- un partenaire IA quand l'automatisation doit rester explicable et validee ;
- un interlocuteur credible pour PME, ASBL et institutions.

## Message principal

Message court :

> Des sites web durables, bien structures et utiles, construits avec une
> expertise PHP/Drupal senior et une IA gardee sous controle.

Message long :

> Nous aidons les PME, ASBL et institutions a creer, refondre et faire evoluer
> des sites web professionnels qui restent maintenables dans le temps. Notre
> approche combine cadrage, architecture de contenu, Drupal, PHP/Symfony/Laravel,
> SEO technique, accessibilite et usages IA concrets, avec validation humaine a
> chaque etape sensible.

## Ton editorial

Le ton doit etre :

- senior, calme et concret ;
- comprehensible par des non-techniciens ;
- transparent sur les limites ;
- oriente decision, pas buzzword ;
- precis sur les livrables ;
- capable de dire quand Drupal, Symfony, Laravel ou l'IA ne sont pas le bon
  choix.

Formulations a privilegier :

- "Nous commencons par clarifier le besoin avant de choisir la technologie."
- "Drupal est pertinent lorsque le contenu, les roles et le multilingue doivent
  rester gouvernables."
- "Symfony ou Laravel peuvent etre plus adaptes pour une application metier
  specifique."
- "L'IA assiste les equipes, elle ne publie pas sans validation humaine."
- "L'audit produit un backlog priorise, pas seulement une liste de problemes."

Formulations a eviter :

- "boostez votre croissance avec l'IA" sans preuve ;
- "agence 360" ;
- "solution revolutionnaire" ;
- "site web pas cher" ;
- "automatisation totale" ;
- "leader" si aucune preuve externe ne le soutient.

## Preuves a produire

Les futures pages doivent contenir au moins deux preuves concretes :

- exemple de livrable d'audit ;
- checklist technique ;
- mini-cas anonymise ;
- exemple de workflow IA avec validation humaine ;
- tableau "Drupal / Symfony / Laravel : quand choisir quoi" ;
- liste de risques evites en refonte ;
- liens internes vers pages services existantes ;
- FAQ decisionnelle.

## Architecture de conversion

Chaque page SEO doit suivre ce schema :

1. H1 oriente besoin, pas techno seule.
2. Introduction qui reformule le probleme du prospect.
3. Sections "quand c'est pertinent" et "quand ce n'est pas le bon choix".
4. Methode de travail.
5. Livrables ou resultats attendus.
6. Preuves ou exemples.
7. Liens internes vers pages proches.
8. CTA unique vers `/contact` ou vers audit selon l'intention.

## Mesure

Avant publication :

- definir mot-cle principal, variantes et intention ;
- verifier l'existence ou non d'une page proche pour eviter cannibalisation ;
- choisir `service`, `page` ou `article` selon la strategie Content Sync ;
- preparer FR et EN ;
- prevoir 2 a 4 liens internes utiles ;
- rediger une meta description differenciee.

Apres publication :

- suivre impressions/clics par URL dans Google Search Console ;
- suivre les requetes qui entrent par les pages generalistes ;
- mesurer les clics vers `/contact` ;
- analyser les formulaires entrants par type de besoin ;
- verifier que les pages IA ne generent pas des leads hors cadre ;
- ajuster le maillage tous les 2 a 3 tickets de contenu.

## Decision strategique

La strategie SEO ne doit pas diluer Drupal. Elle doit replacer Drupal dans une
offre web plus large :

- besoin business : creation, refonte, maintenance, visibilite ;
- decision technique : Drupal, PHP/Symfony/Laravel, integration, IA ;
- preuve senior : methode, audit, arbitrage, validation, documentation ;
- publics : PME, ASBL, institutions.

Le site doit donc devenir visible sur des intentions generalistes, puis
convaincre par sa profondeur technique.
